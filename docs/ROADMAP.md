# 路线图

## 阶段 1：单设备闭环

- 启动 EMQX、InfluxDB、PostgreSQL、Grafana
- 启动 Go 平台服务
- 启动 simulator 或 ESP32 负载节点
- Grafana 查看功率曲线
- HTTP API 下发继电器控制命令
- Docker Compose 一键启动完整演示：已完成
- Docker Compose 健康检查和启动顺序：已完成

## 阶段 2：多设备聚合

- 注册 PV、电池、关键负载、非关键负载
- 接入多个 ESP32
- 展示站点总发电、总负载、净功率、SOC

## 阶段 3：规则调度

- 配置 SOC 上下限
- 配置负载切除阈值
- 调度器自动发布 `set_mode` 和 `set_relay`
- ESP32 回传 command/ack

## 阶段 4：预测优化

- 引入分时电价表：已完成基础版
- 引入光伏/负载预测：已完成本地启发式版本
- 输出未来 24 小时 15 分钟粒度计划：已完成
- 计划跟踪预览：已完成
- 计划建议受控下发：已完成
- 用 Python 服务替换当前 Go 本地计划器
- 接入天气/光照数据
- 将日前计划接入实时调度器，做计划跟踪和偏差修正

## 阶段 5：工程化

- MQTT 用户名/密码：已完成基础版
- MQTT TLS：已完成客户端配置基础版
- 每设备唯一密钥：已完成 HMAC 基础版
- 命令审计持久化：已完成
- 操作审计：已完成基础版
- Redis 实时状态：已完成基础版
- Prometheus 指标：已完成
- Prometheus 告警规则：已完成基础版
